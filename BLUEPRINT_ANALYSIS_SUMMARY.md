## üéØ Critical Context: HR Assistance Tool for Solo HR

**This is NOT an HRIS.** This is a **productivity tool for solo HR professionals** managing 60-100 employees.

### What Solo HR Actually Needs

**Current Reality:**
- One person handling all HR functions
- 30+ hours/week spent on repetitive admin tasks
- Constant interruptions for "Where's my leave balance?" questions
- Manual tracking of compliance deadlines
- Time wasted creating routine documents

**What This Portal Provides:**
- ‚úÖ **Automation** - Alerts, reminders, auto-generation (save 20+ hrs/week)
- ‚úÖ **Self-Service** - Employees find answers themselves (reduce interruptions 80%)
- ‚úÖ **Compliance Safety Net** - Never miss visa/document renewals
- ‚úÖ **Quick Wins** - Generate salary cert in 30 seconds vs 20 minutes

**What This Portal Does NOT Provide:**
- ‚ùå Full payroll calculation engine (use existing payroll service)
- ‚ùå Complex analytics dashboards (Excel exports work fine)
- ‚ùå Enterprise workforce planning (solo HR doesn't need this)
- ‚ùå Advanced talent management (keep it simple)

### Design Philosophy

> **"Build tools that save HR time, not features that add complexity"**

Every feature must answer: **"Does this save solo HR at least 1 hour per week?"**  
If no, it's not a priority.

## Technical Implementation Notes

### Technology Needed

**For Automation:**
- APScheduler (for scheduled tasks)
- Email service (FastAPI-Mail)
- Background job queue (optional: Celery)

**For Document Generation:**
- WeasyPrint or ReportLab (PDF generation)
- Template engine (Jinja2)
- Variable substitution logic

**For Mobile:**
- Tailwind CSS mobile utilities
- Service Workers (offline support)
- Push notification API

**For Workflows:**
- State machine (generic request model)
- Approval routing rules
- Email notification templates

### Database Changes

New tables needed:
- `reimbursements` - Employee expense claims
- `requests` - Generic request workflow
- `announcements` - Company communications
- `profile_completion` - Track missing data
- Enhanced: `notifications` with action buttons

### Security Considerations

All new features must have:
- ‚úÖ Input sanitization (`sanitize_text()`)
- ‚úÖ Role-based access control
- ‚úÖ Parameterized queries
- ‚úÖ Audit logging
- ‚úÖ File upload validation

## Risk Mitigation

### Potential Issues & Solutions

**Issue:** Employees might not adopt new portal  
**Solution:** Make old processes unavailable, provide quick training videos, gamification

**Issue:** Automated emails might fail  
**Solution:** Fallback to manual, monitoring dashboard, retry logic, alternative channels

**Issue:** Document generation might have errors  
**Solution:** Manual override always available, HR review before sending, template validation

**Issue:** Mobile experience might be poor  
**Solution:** User testing with 10 employees, iterative improvements, progressive enhancement

**Issue:** Too many features might overwhelm  
**Solution:** Phased rollout, feature flags, gradual introduction, training materials
